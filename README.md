# CelestialMechanics.jl

**CelestialMechanics.jl** is a high-precision orbital dynamics and perturbation theory framework built in Julia. It provides a robust, type-stable environment for simulating trajectories of celestial bodies and spacecraft around arbitrary central bodies.

The package features a seamless symbolic-numeric interface, allowing researchers to parse analytical Hamiltonians or Lagrange Planetary Equations directly from symbolic systems (Maxima, Mathematica) into optimized Julia code, and integrate them numerically with state-of-the-art solvers.

---

## Main Features

### Propagators
- **Cowell's Method** — Direct Cartesian integration with high-fidelity physical models. Supports first-order (`ODEProblem`) and second-order (`SecondOrderODEProblem`) formulations, enabling symplectic integrators (e.g. `KahanLi8`) for long-term energy conservation.
- **Hamiltonian (Delaunay)** — Integration of action-angle variables from symbolic Hamiltonian inputs. Reads equations generated by external CAS tools at runtime.
- **Lagrange Planetary Equations (LPE)** — Numerical integration of classical Keplerian elements via the Lagrange planetary equations in Gaussian form.
- **N-Body** — Direct N-body propagation with support for both adaptive (Vern9, DOP853) and symplectic (KahanLi8, McAte8) integrators.
- **CR3BP** — Circular Restricted Three-Body Problem propagator with Poincaré section analysis.

### Perturbation Models
- Zonal harmonics J2 through J18
- Tesseral and sectorial harmonics (C_nm, S_nm)
- Solar Radiation Pressure (SRP) with dynamic cylindrical shadow modeling
- N-body third-body gravitational perturbations (arbitrary number of perturbing bodies)

### SPICE Integration
Automated ephemeris retrieval and initial condition generation via NASA's SPICE kernels, with support for arbitrary reference frames and aberration corrections.

### Coordinates and Transformations
- Keplerian elements ↔ Cartesian state vectors
- Delaunay canonical variables ↔ Keplerian elements
- Mean anomaly ↔ true anomaly (robust Newton–Raphson solver)
- Canonical unit normalization (DU, TU, VU) for numerical stability

### Analysis Tools
- Post-processing pipeline for orbital elements from Cartesian trajectories
- Poincaré section computation
- 3D trajectory visualization (GLMakie/CairoMakie)
- Disk-streaming CSV output for long-duration simulations with bounded RAM usage

## Installation

Install directly from GitHub:

```julia
pkg> add https://github.com/guilhermedeolivpaes/CelestialMechanics.jl
```

Once registered in the Julia General Registry (pending), install with:

```julia
pkg> add CelestialMechanics
```

**Requirements:** Julia ≥ 1.10. SPICE kernels are required for ephemeris-based simulations but not for standalone propagation.

## Quick Start

The example below propagates a low Earth orbit with J2 perturbation for 24 hours
using Cowell's method. No SPICE kernels are required for this configuration.

```julia
using CelestialMechanics
using DifferentialEquations
using Unitful, UnitfulAstro
using Unitful.DefaultSymbols

# Initial orbital elements (Example 10.2 - Orbital Mechanics for Engineering Students - Fourth Edition, Curtis, H. D., 2020)
ics = [
    InitialConditions(
        a0 = 8059.0km,
        e0 = 0.17136,
        i0 = 28.0°,
        h0 = 45.0°,
        g0 = 30.0°,
        f0 = 40.0°
    )
]

# Time configuration
t_start  = 0.0u"hr"
t_end    = 24.0u"hr"
step     = 1.0u"minute"
t_vec_u  = t_start:step:t_end
tspan    = (ustrip(u"s", t_start), ustrip(u"s", t_end))
t_vector = ustrip.(u"s", t_vec_u)

# Perturbation model: Earth + J2 only (no SPICE kernels needed)
perturbation_params = create_perturbation_model(:earth,
    j_harmonics = [2]
)

# SpiceInformations: paths are nothing by default — only initial_date is required
spice_info = SpiceInformations(
    initial_date = "2026-01-01T00:00:00"
)

# Propagator options
propagator_opts = PropagatorOptions(
    propagator                   = CowellPropagator(),
    canonical_unit_normalization = true,
    integrator                   = Vern7(),
    abstol                       = 1e-8,
    reltol                       = 1e-8,
    maxiters                     = 10_000_000,
)

# Output directory (created automatically if it does not exist)
output_dir = joinpath(pwd(), "output")

# Run simulation
results = run_simulation(
    ics                 = ics,
    perturbation_params = perturbation_params,
    spice_info          = spice_info,
    tspan               = tspan,
    t_vector            = t_vector,
    propagator_options  = propagator_opts,
    output_directory    = output_dir
);
```

> **Note:** For third-body perturbations (Moon, Sun) or Solar Radiation Pressure,
> valid SPICE kernels must be provided via `SpiceInformations`. Kernels are
> available at [NAIF/JPL](https://naif.jpl.nasa.gov/naif/data.html).
